# Focus Desk: 몰입을 디자인하는 커스텀 디지털 워크스페이스

> **"디지털 환경에서도 물리적 책상처럼 나만의 공간감을 느낄 수는 없을까?"**
> Focus Desk는 이 질문에서 시작된 Electron 기반의 데스크탑 생산성 플랫폼입니다.

---

## 1. 기획 및 분석 (Project Planning)

### 기획 의도: 디지털 유목민을 위한 정착지
현대의 작업자들은 수많은 툴을 사용합니다. 노션으로 문서를 쓰고, 슬랙으로 대화하고, 스포티파이로 음악을 듣고, 크롬으로는 자료를 찾습니다. 하지만 이 과정에서 발생하는 잦은 **창 전환(Context Switching)**은 몰입을 방해하는 가장 큰 요소입니다.

저는 **"흩어진 도구들을 하나의 캔버스에 모으고, 사용자가 가장 편안함을 느끼는 환경을 직접 꾸밀 수 있다면 어떨까?"**라고 생각했습니다. 단순히 기능을 나열하는 것이 아니라, 사용자가 '애정'을 가지고 머물고 싶은 **디지털 공간(Digital Space)**을 만드는 것을 목표로 기획했습니다.


---

## 2. 문제 정의 및 해결 (Problem Solving)

### 😟 문제 1: 도구의 파편화로 인한 집중력 분산
기존에는 할 일을 체크하기 위해 투두 앱을 켜고, 집중 타이머를 위해 폰을 보고, 백색 소음을 위해 유튜브를 틀어야 했습니다.
> **📌 해결책: 올인원 위젯 시스템 (All-in-One Widget System)**
> 자주 사용하는 핵심 도구(Todo, Timer, Calendar, Memo, Browser, Music)를 **위젯(Widget)** 형태로 모듈화하여 하나의 어플리케이션 안에서 실행되도록 통합했습니다. 이를 통해 불필요한 알트탭(Alt+Tab) 빈도를 획기적으로 줄였습니다.

### 😟 문제 2: 천편일률적이고 딱딱한 UI
대부분의 생산성 앱은 '리스트' 형태의 건조한 텍스트 위주입니다. 이는 효율적이지만, 사용자가 '머물고 싶다'는 느낌을 주지는 못합니다.
> **� 해결책: 감성적 몰입 환경 (Emotional Ambience)**
> 생산성에 '감성'을 더했습니다.
> *   **Visual**: 고해상도 배경과 테마 시스템 도입.
> *   **Audio**: '재즈 카페', '비 오는 숲' 등 백색 소음을 직접 믹싱할 수 있는 **Ambience Mixer** 기능을 구현하여 청각적인 공간감을 부여했습니다.

### 😟 문제 3: 브라우저의 유혹
웹 기반의 투두 리스트는 접근성은 좋지만, 새 탭을 여는 순간 유튜브 알고리즘이나 뉴스의 유혹에 빠지기 쉽습니다.
> **📌 해결책: Electron 기반 데스크탑 앱**
> 웹 기술(React)을 사용하되 Electron으로 패키징하여, 브라우저와 분리된 독립적인 실행 환경을 제공했습니다. 이는 심리적으로 "여기는 작업 공간"이라는 경계를 명확히 해줍니다.

## 3. UX/UI 디자인 전략 (UX Strategy)

### 3-1. 글래스모피즘 (Glassmorphism) & 심미성
UI 디자인의 핵심 컨셉은 **'투명함'과 '깊이감'**입니다.
*   **유리 질감**: 위젯 배경에 반투명한 블러(Blur) 효과를 적용하여, 배경화면이 은은하게 비치도록 했습니다. 이는 답답함을 줄이고 공간을 넓어 보이게 합니다.
*   **시각적 계층**: 그림자와 투명도를 조절하여 배경 -> 위젯 -> 컨텐츠로 이어지는 자연스러운 시선의 흐름을 유도했습니다.

### 3-2. 무한한 자유도 (Free-Flow Layout)
대부분의 대시보드 앱이 격자(Grid)에 딱딱하게 맞춰지는 레이아웃을 강요합니다. 하지만 실제 책상은 그렇지 않습니다.
*   **Pixel-Perfect Drag & Drop**: 사용자가 원하는 위치 어디든, 원하는 크기로 위젯을 둘 수 있습니다. 
*   **Layering**: 종이 위에 포스트잇을 붙이듯, 위젯끼리 겹쳐 놓는 것도 허용하여 아날로그적인 사용성을 구현했습니다.

### 3-3. 캄 테크 (Calm Tech) 지향: Focus Mode
집중이 필요할 때 UI는 사라져야 합니다.
*   **Focus Mode 전환**: 버튼 하나로 모든 UI 요소(설정 창, 사이드바 등)가 사라지고, 오직 현재 진행 중인 타이머와 투두 리스트만 남습니다.
*   시각적 노이즈를 최소화하여 사용자가 오직 **'현재의 과업'**에만 몰입할 수 있도록 설계했습니다.

## 4. 기술적 구현 및 최적화 (Technical Details)

### 4-1. Electron + React + Vite 아키텍처
*   **성능 최적화**: 무거운 위젯이 많아질 것을 대비해 `React.memo`와 `useCallback`을 적극 활용하여 렌더링을 최적화했습니다.
*   **Framer Motion**: 부드러운 위젯 등장 및 전환 애니메이션을 위해 Framer Motion을 도입, 끊김 없는 60fps 경험을 제공합니다.

### 4-2. 상태 관리 및 영속성 (Persistence)
*   사용자가 공들여 꾸민 책상 배치가 날아가면 안 됩니다. `electron-store`를 활용하여 창의 크기, 위치, 위젯의 데이터, 오디오 볼륨 설정까지 실시간으로 로컬에 저장하고 복구하는 로직을 구현했습니다.

### 4-3. 데이터 시각화 (Focus Insights)
*   "측정되지 않는 것은 관리될 수 없다"는 원칙 하에, 사용자의 집중 시간을 기록하고 깃허브 잔디(Heatmap) 스타일의 시각화 그래프로 보여주어 성취감과 지속적인 동기를 부여합니다.
